Building the Website

The entire website is a single index.html file. No framework, no build process, no npm install. This was a deliberate choice — maximum simplicity, instant deployment anywhere.

Why a single HTML file?
→ No build step — open directly in browser
→ Deploys instantly to any static host
→ Easy to share, backup, and version control
→ Tailwind CDN handles all styling utilities

Key design rules applied
DM Serif Display italic used only for hero tagline
All interactive elements have cursor-pointer
Transitions capped at 300ms for micro-interactions
SVG icons only — no emoji used anywhere
4.5:1 contrast ratio on all text
Floating navbar with backdrop blur
Outcome: A single index.html file, 660+ lines, fully responsive at 375px → 1440px.

Step 03 of 07
Version Control with Git & GitHub

Version control ensures every change is tracked and reversible. Even for a one-file project, this is essential — it's also the bridge to automatic deployment.

Commands run (in order)
# 1. Initialize a new git repository in the project folder git init git checkout -b main # 2. Create .gitignore to keep sensitive files out of GitHub # (PDFs, Word documents with confidential brand guide) *.pdf, *.docx → NOT pushed to GitHub # 3. Stage and commit the website git add index.html .gitignore git commit -m "Initial commit: personal website for Pourya Gheitarani" # 4. Create GitHub repo + push in one command (GitHub CLI) gh repo create pourya-website --public --source . --remote origin --push

Step 05 of 07
Connecting Supabase

The static site needed a backend for three features: a contact form, a newsletter waitlist, and page visit analytics. Supabase provides a hosted PostgreSQL database with a simple JavaScript API — no server required.

Contact Form
Name, email, message → contact_messages table
Newsletter Waitlist
Email signups → waitlist table (unique emails)
Analytics
Page, referrer, user agent → page_visits table
Database tables created
TableColumnsSecurity
contact_messages id, name, email*, message*, created_at RLS · anon INSERT
waitlist id, email* (unique), created_at RLS · anon INSERT
page_visits id, page, referrer, user_agent, created_at RLS · anon INSERT
Security: Row Level Security (RLS)
RLS was enabled on all three tables so the publishable key (safe to expose in HTML) can only insert data — it can never read, update, or delete rows. Your data is protected.
How the SQL was executed
The Supabase Management API was called directly from Node.js using a Personal Access Token. This avoids needing to open the Supabase dashboard manually — Claude Code ran the SQL on your behalf and confirmed all three tables were created.
POST https://api.supabase.com/v1/projects/{ref}/database/query
Outcome: 3 live tables in Supabase. Forms, waitlist, and analytics all write to the database in real time.

08
Reusable Guide
Repeat This for Any Project

This exact workflow works for any personal website or landing page. Follow this checklist in order every time.

Phase 1 — Design
  • 01 Gather brand guide (PDF, Figma, or notes)
  • 02 Extract: colors, fonts, voice, tagline
  • 03 Define page sections before coding
  • 04 Choose stack (HTML/Tailwind for simple sites)
  • 05 Run /ui-ux-pro-max in Claude Code for guidance
Phase 2 — Build
  • 06 Build index.html with Tailwind CDN
  • 07 Test responsive at 375px, 768px, 1440px
  • 08 Check accessibility: alt text, focus states, contrast
Phase 3 — Ship
  • 09 git init in project folder
  • 10 Create .gitignore (exclude sensitive files)
  • 11 gh repo create → push to GitHub
  • 12 npx vercel --prod → site is live
  • 13 Write README.md for future reference
Phase 4 — Connect Backend
  • 14 Create Supabase project, get publishable key
  • 15 Create tables with RLS + anon insert policy
  • 16 Add Supabase JS CDN + createClient() to HTML
